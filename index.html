How it works

It rewrites the agent.
Not just its notes.

Other systems bolt memory onto an unchanged agent. agent-brain goes deeper — it routes each lesson to either context (recall when needed) or definition (who the agent fundamentally is). High-signal learnings get promoted into the agent's core, permanently.

Four modes

Four operations.
One direction: better.

Click any mode to see it in action. Each command is a different way to push the agent's core forward — from a single explicit lesson to a full self-audit that rewrites what the agent fundamentally is.

agent-brain improve

Improve

Run after any session. Reflects on corrections, loops, and interventions — then promotes the highest-signal learnings straight into the agent's definition.

↗
agent-brain learn <something>

Learn

Explicitly push a lesson into the agent. The router decides whether it belongs in context or in core. Either way, it sticks — permanently.

↗
agent-brain absorb <path>

Absorb

Point at another agent or skill. It maps what it doesn't already know, surfaces conflicts, and selectively integrates the delta into its own definition.

↗
agent-brain dream

Dream

A full self-audit. Surfaces stale context, gaps in the definition, and patterns worth promoting to core. Leaves the agent leaner and sharper than it found it.

Storage architecture

Two tiers.
One direction.

Learnings don't all belong in the same place. The PARA system separates what the agent looks up from what the agent is — and over time, the best context gets promoted into the core.

P Active work

Projects

Short-horizon work with a clear outcome. Closed projects move to Archives automatically.

A Ongoing standards

Areas

Responsibilities that persist over time. Coding standards, team conventions, agent identity traits.

R Reference material

Resources

Useful things to know but not act on. Domain knowledge, API docs, research notes.

A Completed work

Archives

Inactive items kept for reference. Past projects, superseded patterns, deprecated constraints.
